# vertex_leaders.lsp

AutoLISP-скрипт для AutoCAD, автоматизирующий простановку координатных выносок по вершинам полилинии и параллельных размеров по её сегментам.

## Возможности

- Автоматическая простановка номеров вершин и их координат (X/Y) двумя способами на выбор:
  - **Мультивыноска (MLEADER)** — текст вида `X= +3373.14` / `Y= +458.40` с полкой без стрелки;
  - **Атрибутивный блок «XY»** — вставляется точкой прямо в вершину полилинии, без линии-выноски.
- Автоматическое создание/импорт нужных стилей оформления из чертежа-шаблона:
  - стиль мультивыноски **«Координаты»**;
  - текстовый стиль **«Д-431»**;
  - размерный стиль **«ISO-500 пустой COORD»**;
  - блок **«XY»** с атрибутами X и Y.
  Если импорт из шаблона не удался — скрипт работает на базе стиля `Standard`, вручную выставляя все параметры оформления.
- Построение параллельных (aligned) размеров по каждому сегменту полилинии, смещённых **под** полилинию:
  - если сегмент достаточно длинный — значение длины выводится прямо на линии размера;
  - если сегмент короткий — значение выносится отдельной мультивыноской в сторону.
- Сквозная нумерация вершин между несколькими полилиниями в рамках одного сеанса AutoCAD, с возможностью продолжить нумерацию или начать заново.
- Экспорт итоговой таблицы координат в **CSV** (Excel) и/или **RTF** (Word).

## Требования

- AutoCAD 2018 и новее (используется ActiveX/VLA, `AddMLeader`, `AddDimAligned`).
- Файл чертежа-шаблона со стилями и блоком (см. раздел «Настройка шаблона» ниже).

## Установка

1. Скачайте `vertex_leaders.lsp` в рабочую папку (например, `D:/Projects/Lisp/LISP/`).
2. В AutoCAD выполните команду `APPLOAD`, укажите путь к файлу и нажмите «Загрузить».
3. При успешной загрузке в командной строке появится:

```
+---------------------------------------------------+
|  vertex_leaders.lsp успешно загружен.              |
|  Доступные команды:                                |
|    VERTEXLEADERS  — координаты + размеры + экспорт |
|    CHECKSTYLE     — проверить/импортировать стили  |
|    RESETVERTEXNUM — сбросить счётчик нумерации     |
+---------------------------------------------------+
```

> Чтобы скрипт подгружался автоматически при каждом запуске AutoCAD, добавьте его в `acad.lsp` / `acaddoc.lsp` или пропишите в списке автозагрузки (`APPLOAD` → «Содержимое»).

## Настройка шаблона

В начале функции `c:VERTEXLEADERS` задан путь к чертежу-шаблону:

```lisp
(setq template-path "D:/Projects/Lisp/LISP/++РАМКА++.dwg")
```

Отредактируйте этот путь под своё расположение файла. В шаблоне должны быть заранее настроены:

| Объект | Имя | Требования |
|---|---|---|
| Стиль мультивыноски | `Координаты` | без стрелки, горизонтальная полка, текстовый стиль «Д-431», маска фона |
| Текстовый стиль | `Д-431` | используется для координат, номеров вершин и подписей размеров |
| Размерный стиль | `ISO-500 пустой COORD` | без стрелок, без выносных линий, текстовый стиль «Д-431» |
| Атрибутивный блок | `XY` | атрибуты с тегами `X` и `Y` (латиница) — допускаются также кириллические омоглифы `Х`/`У` |

## Команды

### `VERTEXLEADERS`

Основная команда. Порядок работы:

1. Скрипт проверяет/импортирует текстовый стиль, стиль мультивыноски и размерный стиль.
2. Запрашивает выбор полилинии на чертеже.
3. Запрашивает способ вывода координат: **Мультивыноска** или **Блок**.
4. Для каждой вершины полилинии создаёт:
   - подпись координат (мультивыноской или блоком «XY»);
   - номер вершины (замаскированный MTEXT).
5. Для каждого сегмента полилинии строит параллельный размер со значением длины под полилинией.
6. Предлагает экспортировать накопленную таблицу координат в CSV и/или RTF.

```

```

### `CHECKSTYLE`

Служебная команда — только проверяет/импортирует текстовый стиль «Д-431» и стиль мультивыноски «Координаты» из шаблона, ничего не рисуя на чертеже. Полезна для диагностики перед запуском `VERTEXLEADERS`.

### `RESETVERTEXNUM`

Сбрасывает сквозной счётчик нумерации вершин и накопленную таблицу экспорта без перезапуска AutoCAD. После сброса следующий запуск `VERTEXLEADERS` начнёт нумерацию с №1 без дополнительного вопроса.

## Формат вывода координат

- Координаты форматируются с обязательным знаком и двумя знаками после запятой: `+3373.14`, `-458.40`.
- Длины сегментов форматируются без знака, с двумя знаками после запятой: `1234.50`.
- В режиме «Мультивыноска» текст выглядит как:
  ```
  X= +3373.14
  Y= +458.40
  ```
- В режиме «Блок» префиксы `X=`/`Y=` формируются скриптом и записываются прямо в атрибуты блока (см. скриншот ниже).

## Экспорт таблицы координат

По завершении простановки выносок скрипт предлагает выбрать формат экспорта:

- **Excel** — CSV-файл с разделителем `;` (корректно открывается в Excel с русской локалью), колонки `№;X;Y`;
- **Word** — RTF-файл с готовой таблицей (заголовок, границы ячеек), открывается в Word напрямую, без COM-автоматизации;
- **Оба** — оба файла сразу;
- **Нет** — пропустить экспорт.

Таблица накапливается **глобально** в переменной `*VL:ALL-EXPORT-DATA*` — то есть при обработке нескольких полилиний за один сеанс в экспорт попадут координаты вершин всех обработанных полилиний с их сквозными номерами.

## Известные особенности и ограничения

- Прямое создание объекта `MLEADERSTYLE` через `entmake` ненадёжно, поэтому стиль импортируется из шаблона путём временной вставки блока с последующим удалением (`-INSERT` / `ERASE`).
- Атрибуты блока `XY`, оформленные как **поле (Field)**, могут быть перезаписаны при следующей регенерации чертежа — скрипт выводит диагностическое предупреждение, если обнаруживает это.
- Поддерживаются полилинии типов `LWPOLYLINE`, `POLYLINE`, `3DPOLYLINE`.
- Счётчик нумерации вершин (`*VL:LAST-VERTEX-NUM*`) хранится в памяти сеанса AutoCAD и сбрасывается только командой `RESETVERTEXNUM` или перезапуском AutoCAD.
- Если предыдущая обработанная полилиния была удалена с чертежа, при следующем запуске `VERTEXLEADERS` счётчик и накопленные данные экспорта сбрасываются автоматически.

## Структура файла

| Раздел | Назначение |
|---|---|
| `vl:fmt-coord`, `vl:fmt-length` | Форматирование чисел в строки |
| `vl:mleader-style-exists-p`, `vl:dimstyle-exists-p` | Проверка наличия стилей в чертеже |
| `vl:ensure-text-style` | Создание текстового стиля, если отсутствует |
| `vl:ensure-mleader-style-from-template`, `vl:ensure-dimstyle-from-template`, `vl:ensure-coord-block-from-template` | Импорт стилей/блока из чертежа-шаблона |
| `vl:create-mleader` | Создание мультивыноски через ActiveX |
| `vl:create-mtext-masked` | Создание замаскированного MTEXT |
| `vl:insert-coord-block` | Вставка атрибутивного блока «XY» в вершину |
| `vl:create-parallel-dim-with-label` | Построение параллельного размера по сегменту с подписью длины |
| `vl:get-polyline-vertices` | Извлечение списка вершин полилинии |
| `vl:export-csv`, `vl:export-rtf`, `vl:offer-export` | Экспорт таблицы координат |
| `c:VERTEXLEADERS` | Главная команда |
| `c:CHECKSTYLE` | Диагностика стилей |
| `c:RESETVERTEXNUM` | Сброс счётчика |
