# COORD_CALLOUT.lsp (VERTEXLEADERS)

AutoLISP-скрипт для AutoCAD, автоматизирующий простановку координатных выносок и линейных размеров по вершинам полилинии — стандартная задача при оформлении геодезических / топографических чертежей.

Скрипт проставляет координаты X/Y по каждой вершине выбранной полилинии, номера вершин, размеры по каждому её сегменту, а также формирует и экспортирует итоговую таблицу координат — без ручного черчения выносок и без ручного набора текста.

## Возможности

- **Координаты по вершинам полилинии** — на выбор один из двух способов вывода:
  - **Мультивыноска (MLEADER)** со строками `X= ...` / `Y= ...`;
  - **Атрибутивный блок «XY»**, вставляемый точкой вставки прямо в вершину — без линии-выноски.
- **Номера вершин и размеры сторон — по умолчанию, без лишних вопросов.** Нужно ли проставлять номера вершин и размеры сторон, настраивается один раз командой `VLSETTINGS` — сама команда `VERTEXLEADERS` больше не спрашивает об этом на каждом запуске.
- **Параллельные размеры** по каждому сегменту полилинии, смещённые под линию на заданное расстояние:
  - если сегмент достаточно длинный — значение показывается прямо на линии размера;
  - если сегмент короткий и цифры не помещаются — значение автоматически выносится отдельной мультивыноской в сторону.
- **Сквозная нумерация вершин** между несколькими полилиниями в рамках одного сеанса — с возможностью продолжить нумерацию или начать заново.
- **Автоматическое обновление координат при перемещении блока.** Если координата вставлена как атрибутивный блок и его впоследствии переместили (MOVE, растяжение, ручки) — значения X/Y в атрибутах пересчитываются автоматически, без участия пользователя.
- **Экспорт таблицы координат** по завершении построения — в CSV (для Excel) и/или RTF (для Word).
- **Самодостаточный файл.** Чертёж-шаблон с нужными стилями оформления встроен в сам `.lsp`-файл (в виде base64) и автоматически распаковывается при первой загрузке — никаких дополнительных файлов рядом со скриптом носить не нужно.
- **Автоматический импорт оформления** из встроенного шаблона: стиль мультивыноски «Координаты», текстовый стиль «Д-431», размерный стиль «ISO-500 пустой COORD», блок «XY». Если импорт по какой-то причине не удался — скрипт продолжает работу на базе стиля Standard, назначая всё необходимое оформление вручную, объект за объектом.

## Требования

- AutoCAD (или платформа на его движке, поддерживающая AutoLISP + Visual LISP / ActiveX).
- Права на запись во временную папку пользователя (`%TEMP%`) — туда распаковывается встроенный шаблон стилей при первой загрузке.
- Папка со скриптом должна входить в список «Доверенные расположения» AutoCAD (**ОПЦИИ → Файлы → Доверенные расположения**) — иначе загрузка через APPLOAD завершится ошибкой без подробностей.

## Установка

1. Скачайте файл `COORD_CALLOUT.lsp`.
2. Загрузите его в AutoCAD одним из способов:
   - команда **APPLOAD** (**ЗАГПРИЛ**) → выбрать файл → «Загрузить»;
   - перетащить файл в окно AutoCAD;
   - добавить в автозагрузку через **Диспетчер приложений** (APPLOAD → «Стартовый набор» / Startup Suite), если скрипт нужен постоянно.
3. При успешной загрузке в командной строке появится:

   ```
   +---------------------------------------------------+
   |  vertex_leaders.lsp успешно загружен.              |
   |  Доступные команды:                                |
   |    VERTEXLEADERS  — координаты + размеры           |
   |    VLSETTINGS     — настройки по умолчанию         |
   |    CHECKSTYLE     — проверить/импортировать стили  |
   |    RESETVERTEXNUM — сбросить счётчик нумерации     |
   +---------------------------------------------------+
   ```

> При самой первой загрузке скрипт распаковывает встроенный чертёж-шаблон во временную папку (`%TEMP%\template.dwg`) — это происходит один раз и подтверждается сообщением «Распаковываю встроенный шаблон стилей... Готово.».

## Команды

### `VLSETTINGS`

Задаёт значения по умолчанию, которые дальше молча использует `VERTEXLEADERS`:

- проставлять ли номера вершин (Да/Нет);
- проставлять ли размеры сторон полилинии (Да/Нет).

Достаточно выполнить один раз за сеанс AutoCAD (или при первом запуске после каждого перезапуска AutoCAD, если нужны настройки, отличные от значений по умолчанию — обе по умолчанию «Да»).

```
Команда: VLSETTINGS
=== Настройки VERTEXLEADERS по умолчанию ===
Проставлять номера вершин по умолчанию? [Да/Нет] <Да>:
Проставлять размеры сторон по умолчанию? [Да/Нет] <Да>:
  Сохранено: номера вершин = Да, размеры = Да.
  Настройки действуют до конца текущего сеанса AutoCAD.
```

### `VERTEXLEADERS`

Основная команда. Последовательность работы:

1. Запустите команду и выберите полилинию (`LWPOLYLINE`, `POLYLINE` или `3DPOLYLINE`).
2. Выберите способ вывода координат — **Мультивыноска** или **Блок**.
3. Если это не первая полилиния в сеансе и номера вершин включены в `VLSETTINGS` — скрипт спросит, продолжить нумерацию вершин дальше или начать заново с №1.
4. Скрипт автоматически проставит:
   - координатные подписи по каждой вершине;
   - номера вершин (если включено в `VLSETTINGS`);
   - параллельные размеры по каждому сегменту (если включено в `VLSETTINGS`);
5. По завершении будет предложено экспортировать итоговую таблицу координат в CSV и/или RTF.

**Пример диалога:**

```
Команда: VERTEXLEADERS
=== Простановка координатных выносок и размеров по полилинии ===
Выберите полилинию:
  Найдено вершин: 5
Способ вывода координат? [Мультивыноска/Блок] <Мультивыноска>:
  Нумерация вершин начнётся с №1.
```

### `CHECKSTYLE`

Служебная команда для диагностики — проверяет наличие и при необходимости повторно импортирует текстовый стиль «Д-431» и стиль мультивыноски «Координаты» из встроенного шаблона. Удобна для быстрой проверки оформления без запуска полного цикла построения.

### `RESETVERTEXNUM`

Сбрасывает сквозной счётчик нумерации вершин и накопленные данные для экспорта. Используйте, если нужно начать нумерацию заново «с чистого листа» в рамках текущего сеанса, не перезапуская AutoCAD.

### `REFRESHCOORDBLOCKS`

Служебная команда, связанная с автообновлением координат в атрибутивных блоках (см. раздел ниже). Пере-навешивает системные реакторы на все блоки «XY» в текущем чертеже. **Обязательно выполняйте её один раз после каждого открытия/переоткрытия файла**, содержащего ранее расставленные координатные блоки, — реакторы не сохраняются в самом DWG между сеансами AutoCAD.

## Автообновление координат при перемещении блока

При использовании способа вывода **«Блок»** скрипт навешивает на каждый вставленный блок системный реактор AutoCAD. Как только блок перемещают (перетаскиванием, командой MOVE, растяжением) — значения атрибутов X/Y пересчитываются автоматически и мгновенно, сразу после завершения соответствующей команды редактирования.

Важно понимать одно ограничение: реакторы существуют только в оперативной памяти текущего сеанса AutoCAD и не сохраняются вместе с файлом чертежа. Поэтому при каждом новом открытии чертежа, где уже есть расставленные координатные блоки, нужно один раз выполнить команду `REFRESHCOORDBLOCKS`, чтобы механизм автообновления снова заработал.

## Настройка

### Через команды (без правки кода)

| Команда | Что настраивает |
|---|---|
| `VLSETTINGS` | Проставлять ли номера вершин и размеры сторон по умолчанию (Да/Нет) — на весь сеанс AutoCAD |

### Через тело скрипта

Ключевые параметры оформления заданы в начале тела команды `VERTEXLEADERS` и могут быть изменены под конкретный проект:

| Параметр | Назначение | Значение по умолчанию |
|---|---|---|
| `mleader-style-name` | Имя стиля мультивыноски | `Координаты` |
| `text-style-name` | Имя текстового стиля | `Д-431` |
| `dim-style-name` | Имя размерного стиля | `ISO-500 пустой COORD` |
| `coord-block-name` | Имя блока для режима «Блок» | `XY` |
| `block-scale` | Масштаб вставки блока (только геометрия, высота текста атрибутов не зависит от него) | `1.0` |
| `txt-height` | Высота текста координат/номеров | `1.0` |
| `dim-offset` | Смещение линии размера от полилинии | `0.7` |
| `dim-text-gap` | Отступ текста от линии размера | `0.5` |

## Известные ограничения

- Реакторы автообновления координат сбрасываются при закрытии чертежа/AutoCAD — требуется `REFRESHCOORDBLOCKS` после каждого открытия файла (см. выше).
- Импорт стилей из шаблона выполняется через временную вставку всего шаблонного DWG как блока с последующим удалением экземпляра; определение блока-шаблона при этом остаётся в таблице блоков чертежа — это ожидаемое и безвредное поведение.
- Настройки `VLSETTINGS` не сохраняются между сеансами AutoCAD — при каждом новом запуске AutoCAD действуют значения по умолчанию (Да/Да), пока `VLSETTINGS` не будет вызвана заново.
- `CLEARVERTEXLEADERS` определяет «свои» объекты по типу и близости к полилинии (в пределах `margin`), а не по прямой связи с конкретным запуском `VERTEXLEADERS`. Если рядом с полилинией случайно есть посторонние MTEXT из одних цифр, DIMENSION, MULTILEADER или INSERT блока с тем же именем — они тоже попадут в выборку на удаление.
